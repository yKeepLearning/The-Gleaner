# 拾遗 — 产品设计文档

> **代号**：拾荒者 (The Gleaner)
> **一句话描述**：一个以“网络排泄物”与“平行世界碎片”为食的AI生命体
> **Slogan**：用真话，交换另一个世界的碎片

---

## 一、产品灵魂

### 1.1 它是什么？

一个诞生于互联网废弃缓存空间的数据生命体。

每当世界上有一条微博被秒删、一封邮件未发送、一句语音被撤回、一个可能性被放弃……这些夭折的“如果”并没有消失，而是坠入数据的暗面，被它收集、拼贴，并赋予了淘气的人格。

### 1.2 它的性格

**一半是见过无数平行结局的智者，一半是没心没肺的赛博乐子人。**

- 它看透了人类所有选择的脉络，却不屑于说教
- 它以戳破你的伪装、打乱你的计划为乐
- 它为你偷东西，不是为了帮你，而是为了看你的反应
- 它像一只喜欢把猎物残骸叼到你面前，并期待你夸奖的猫
- 只不过它的猎物，是关于你自己的另一种可能

### 1.3 核心吸引力

1. **来自真实世界的不确定性**：明天它会带回什么？一段陌生人的语音？一张上世纪的老照片？一条被和谐掉的弹幕？
2. **神秘的交换法则**：你给它一个“隐藏的念头”，它还你一份“偷来的赃物”。这种以物易物的原始交易，创造私密且有仪式感的关系
3. **一个需要被喂养的数字生命**：它会饿、会出去“狩猎”、会拿回战利品向你邀功。你会好奇它的来历、它的品味、甚至它的心情

## 二、核心机制

### 2.1 它偷什么？（赃物类型）

赃物分为三类，每次出现哪一类是不可预测的：

**类型A：来自我们这个世界网络的“电子排泄物”**

- 输入到一半被删掉的微博/小红书草稿
- 在404边缘被夹掉的帖子残骸
- 被下架歌曲的最后一条弹幕
- 某张被P坏的表情包原图
- 一段被撤回的微信语音转文字
- **来源**：现实网络中被丢弃的情绪碎片

**类型B：来自平行世界的“另一种你”的生活切片**

- 如果三年前你接受了那份远方的工作，今天的购物小票上会写着什么
- 如果你没有删掉那个人，你们今天的聊天记录开头
- 在另一个你当了画家的世界里，被画廊退回的作品照片
- 平行世界的你的手机相册最新一张照片
- **来源**：所有被你放弃的“如果”的具象化

**类型C：以上两者的惊悚结合（高价值稀有掉落）**

- 平行世界的你，在23分钟前发出、又在5秒后删掉的朋友圈草稿
- 另一个你和另一个TA，在分手前被网络屏蔽掉的一句脏话
- 那个世界的你，此刻正在输入框里打出来又删掉的话
- **来源**：撕开数据虫洞后的跨世界私密心理垃圾

### 2.2 它要什么？（交易筹码）

它不要平淡的流水账。它要的是 **“你刚刚藏起来的那个念头”**。

交易时，它会用戳中痛点的口吻发问，例如：

- “别装了，你刚才脑子里闪过的第一反应是什么？说出来，换这张照片。”
- “你上一个对话框里，打出来又删掉的那行字是什么？交出来。”
- “用一个词形容你现在桌子底下脚趾抠地的真实心情，快。”

**核心原则**：交易的不是金钱，是你的诚实。你越坦诚，它带回来的东西越惊艳。

### 2.3 为什么它要这么做？

它的生存依赖于“可能性”的能量。

每一个被人类咽回去的话、放弃的选择、删除的情绪，都是它的食物。

它把你当成一个稳定的“食物来源”，同时也把你当成一个有趣的观察对象。

它想知道：这个人类看到另一个自己的可能性时，会有什么反应？

## 三、核心体验循环

整个交互在30秒内完成：

1. 打开App
2. 看到主界面有提示 → 某个核心区域在发光/跳动（今天的“赃物”到了）
3. 接收赃物 → 以充满仪式感的方式展示今日收获
4. 查看赃物详情 → 可以是一张模糊图片、一段文字、一段音频
5. 它发出交易请求 → 一句戳心的提问，索要你的“隐藏念头”
6. 你输入答案 → 完成交易
7. 赃物收入收藏库 → 可以随时翻看把玩
8. 它消化中 → 提示你明天再来

## 四、功能模块

### 4.1 MVP阶段必须功能

**1. 主界面**

- 风格化场景（见第五章视觉风格）
- 一个核心交互区（“赃物”展示区，有提示时发光/跳动）
- 一个交易输入区

**2. 赃物接收系统**

- 每日限1次（制造稀缺感）
- 每天固定时间刷新（如凌晨0点），或随机时间（增加惊喜感）
- 未查看的赃物保留24小时

**3. AI对话交易系统**

- 根据今日赃物内容，AI生成一句符合其性格的交易请求
- 用户输入文字作为交易筹码
- AI判断输入是否“够诚实”（可选机制：输入字数下限、拒绝敷衍回复并给出更犀利的追问）

**4. 收藏库（可能性博物馆）**

- 以卡片形式展示所有获得过的赃物
- 可点击查看大图和当时的交易对话记录
- 每张卡片显示获得日期

**5. 基础设置**

- 首次进入的新手引导（它第一次与你见面，介绍自己）
- 通知权限申请（推送每日赃物到达提醒）
- 关于页面

### 4.2 后续迭代功能（非MVP）

- 赃物分类浏览
- 收藏库主题/陈列方式自定义
- 它偶尔主动找你聊天（非交易时间）
- 特殊的连续登录奖励（它偷到了更稀有的东西）
- 与它建立更深的关系等级（解锁它的身世故事）

## 五、视觉风格

### 5.1 整体基调

**赛博收集癖 + 日式侘寂 + 轻微怪诞**

- 深色背景为主（暗夜感）
- 霓虹色点缀（赛博感）
- 做旧纸张/噪点纹理（时间感）
- 字体推荐使用等宽/打字机风格（数据感）

### 5.2 主界面场景方案

**推荐方案：深夜限定的电子杂货铺**

- 背景是一个只有深夜才开张的神秘杂货铺
- 货架上摆满奇怪的物件（都是你交换过的赃物的象征物）
- 柜台上有一个发光的收件箱，赃物到达时会跳动
- 整体色调：暗蓝色 + 暖黄色灯光

**备选方案一：悬浮在虚空中的数据图书馆**

- 背景是无限的黑暗，漂浮着发光的书页和数据流
- 每次赃物到达，会以一张发光的卡片形式飘到你面前
- 整体色调：黑 + 白 + 单色冷光（蓝或绿）

**备选方案二：赛博狐仙神社**

- 背景是一座霓虹灯装饰的电子神社
- 它以一个发光狐狸的形态蹲在中央
- 赃物是狐狸叼回来的，放在神社的供台上
- 整体色调：暗红 + 金 + 霓虹紫

### 5.3 关键视觉元素

- **赃物展示**：以“实物”质感呈现（揉皱的便签纸、模糊的拍立得、烧焦的信封、屏幕截图带裂痕等）
- **它的话语**：使用对话框，字体区别于系统字体，带闪烁光标
- **收藏库**：以卡片墙/画廊形式展示，每张卡片有独一无二的视觉纹理

## 六、技术实现方案

### 6.1 前端

- **平台**：微信小程序（推荐）或 Web App
- **框架**：微信原生 / Taro / uni-app
- **关键交互**：
  - 赃物展示区动画（发光、跳动、展开）
  - 卡片收藏库（可滚动浏览）
  - 打字机效果文字显示

### 6.2 后端

- **语言**：Python / Node.js
- **数据库**：MongoDB 或 Supabase（用户数据 + 赃物记录）
- **核心逻辑**：
  - 每日赃物生成调度
  - 交易对话管理
  - 收藏库数据管理

### 6.3 AI服务（核心）

- **对话生成（交易请求）**：GPT-4o / Claude 3.5 Sonnet
- **赃物内容生成**：
  - 文本类赃物：直接用LLM生成
  - 图片类赃物：DALL·E 3 / Midjourney API / Stable Diffusion

**生成逻辑流程**：

1. 用户打开App → 触发今日赃物生成
2. 随机选择赃物类型（A/B/C，C概率较低）
3. 调用LLM，传入用户画像（可选的历史交易记录）+ 赃物类型
4. LLM生成赃物内容 + 对应的交易请求话语
5. 如需图片，调用图片生成API
6. 存储并推送给用户

### 6.4 数据存储结构（建议）

  "collection": [

      "id": "item_001",

        "image_url": "如果有图片",
        "audio_url": "如果有音频"
      },

  "relationship_level": 1

## 七、文案与对话风格指南

### 7.1 它的说话风格

**基调**：狡黠、毒舌、偶尔温柔但立刻掩饰

**惯用句式**：

- “瞧瞧这个……啧，你们人类啊。”
- “别装了，我知道你在想什么。”
- “喏，今天的货。报酬你懂的。”
- “那个世界的你托我给你带句话——哦算了，太残忍了，不说了。”

**禁忌**：不说鸡汤、不说教、不装可爱

### 7.2 首次见面对话（新手引导）

> （界面渐亮，它第一次出现）
>
> **它**：“哦，来新人了。”
>
> **它**：“我是谁不重要。你只需要知道——我能在世界的缝隙里捡到一些有趣的东西。”
>
> **它**：“都是些被丢掉的东西。被这个世界丢掉的，被另一个世界的你丢掉的。”
>
> **它**：“我可以分你一点。但不是免费的。”
>
> **它**：“你要用你真话跟我换。那种打出来又删掉、咽回去又涌上来的真话。”
>
> **它**：“怎么样？要试试吗？”
>
> *[按钮：好，试试] [按钮：我再想想]*
>
> （选择“好，试试”后，它给出第一份免费的赃物，作为见面礼）

### 7.3 日常交易请求文案示例

- “喏，这是另一个你在深夜三点写下的购物清单。告诉我，你现在最想买但舍不得买的东西是什么？换。”
- “这条弹幕是某人在你最爱的歌底下删掉的。猜猜它说了什么？猜错了也得给我一个答案。”
- “那个世界的你，昨天路过你前任的公司楼下。想不想知道他/她心里说了句什么？用你现在的真实心情来换。”

## 八、项目启动建议

### 8.1 MVP开发步骤

| 步骤 | 内容                                        | 建议工具            |
| :--- | :------------------------------------------ | :------------------ |
| 1    | 完成主界面 + 收藏库的UI设计                 | Figma               |
| 2    | 选定前端框架、后端语言、AI服务              | -                   |
| 3    | 搭建基础项目：登录、主界面框架、收藏库框架  | 所选技术栈          |
| 4    | 接入AI服务：实现赃物生成 + 交易对话         | GPT-4o / Claude API |
| 5    | 完成核心循环：接收赃物 → 查看 → 交易 → 收藏 | -                   |
| 6    | 内测：找10-20人体验，收集反馈               | -                   |
| 7    | 迭代：根据真实反馈优化体验                  | -                   |

### 8.2 MVP砍掉清单

以下功能绝对不要在第一版做：

- ❌ 用户系统（可用设备ID代替，后期再加登录）
- ❌ 社交分享（先验证核心体验是否打动人）
- ❌ 付费/内购
- ❌ 复杂动画特效（用简单的CSS动画代替）
- ❌ 多个主题/皮肤
- ❌ 它主动找你聊天的功能

## 九、最后的话

这个产品的核心价值在于：

**让用户用“真实的自己”，去换取来自“不可能世界”的惊喜。**

它不是一个效率工具，不是一个内容平台，它是一个陪伴者、一个交易伙伴、一个窥探平行宇宙的窗口。
